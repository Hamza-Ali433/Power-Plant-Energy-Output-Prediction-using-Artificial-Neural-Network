# Power Plant Energy Output Prediction using Artificial Neural Network (PyTorch)

## Overview

This project predicts the electrical energy output (PE) of a Combined Cycle Power Plant using an Artificial Neural Network (ANN) built with PyTorch.

The goal is to learn the relationship between environmental conditions and the amount of power generated by the plant. Instead of using traditional machine learning algorithms, this project implements a fully connected neural network from scratch using PyTorch.

---

## Problem Statement

Power plants are affected by changing environmental conditions such as temperature, atmospheric pressure, humidity, and exhaust vacuum. These variables directly influence the amount of electricity that can be generated.

The objective of this project is to build a regression model that accurately predicts the net hourly electrical energy output based on these operating conditions.

---

## Dataset

The dataset contains **9,568 observations** with **4 input features** and **1 target variable**.

### Features

- **AT** – Ambient Temperature
- **V** – Exhaust Vacuum
- **AP** – Ambient Pressure
- **RH** – Relative Humidity

### Target

- **PE** – Net Hourly Electrical Energy Output

Dataset Summary:

- Total Records: **9,568**
- Features: **4**
- Target: **1**
- Missing Values: **None**

## Project Workflow

### 1. Data Loading

The dataset is loaded using Pandas and inspected for missing values, data types, and overall structure.

### 2. Data Preparation

- Selected input features (AT, V, AP, RH)
- Selected target variable (PE)
- Split the dataset into training and testing sets
- Applied StandardScaler to normalize the input features

### 3. Tensor Conversion

The scaled data was converted into PyTorch tensors before training the neural network.

### 4. DataLoader

PyTorch's `TensorDataset` and `DataLoader` were used to create mini-batches for efficient training.

Hidden Layer 2

- 6 Neurons
- ReLU Activation

Output Layer

- 1 Neuron

Loss Function

Optimizer

- Adam Optimizer

Training Epochs

- 50

---

## Model Checkpoint

During training, the model continuously monitors the validation loss.

Whenever the validation loss improves, the model weights are saved automatically as:

```
best_loss.pt
```

This ensures that the best-performing model is used for final evaluation.

## Interpretation of Results

The model achieved an **R² score of approximately 0.93**, which means it explains around **93% of the variation** in the plant's energy output.

The training and testing losses are very close, indicating that the model generalizes well and does not show significant signs of overfitting.

A comparison between actual and predicted values also shows that the network is able to estimate power output with good accuracy.

Example predictions:

| Actual | Predicted |
|---------|----------:|
| 433.27 | 434.50 |
| 438.16 | 435.71 |
| 458.42 | 461.05 |
| 480.82 | 476.54 |
| 456.78 | 456.40 |

Most predictions remain very close to the actual values, demonstrating the effectiveness of the trained ANN.
